Transfer Learning in Bone Morphology Study for Better Hip Fracture Assessment

Biomedical Image Analysis, Vision Transformer & Applied Machine Learning

Jun 2023 -- Jun 2024

This study was done under the supervision of Dr. Mahmuda Naznin , Professor, Ex-Dept. Head, CSE, BUET and Dr. Tanvir Faisal, Assistant Professor, ME, University of Louisiana at Lafayette. The manuscript is currently in submission. The study focuses on predicting the fracture risk index from the voxelized bone data. Applied machine learning techniques are used to predict the stress and strain distributions over the nodes of the bone. The stress, strain and bone density values are then plotted using the rgb channels respectively to get a visual representation of the values. Different views are generated by rotating the bone. A vision transformer is trained on the different views to find the best performing view angle. The vision transformer is modified to find the layer architecture with highest accuracy and less computing time. This method has an accuracy of almost 95% and works better than previous methods of prediction and PointNet-based classification.

Improvement on Hip Fracture Assessment Pipeline

Multimodal Learning, 3D Image Analysis, Biomedical Image Analysis, Vision Transformer & Applied Machine Learning

Nov 2024 -- Present

This study was done under the supervision of Dr. Mahmuda Naznin , Professor, Ex-Dept. Head, CSE, BUET and Dr. Tanvir Faisal, Assistant Professor, ME, University of Louisiana at Lafayette. The current pipeline for fracture risk assessment stands as: voxelization -> csv data -> regression -> image data -> classification with vision transformer -> Fracture Risk index. However, each of these steps is very time intensive and resource hungry, impractical for everyday medical use. Also we left out patients physiological features like - age, race, sex etc. We plan to use multimodal architecture to shorten this pipeline and integrate those features. We will also employ some PEFT techniques to bring the computational time and resources down to practical levels. The new pipeline will have 3 stages. Stage 1 and 2 can be run parallely and thus save time. Stage 1: bone qct slices -> grayscale value arrays -> pixel positional encoding -> image encoding -> vision transformer based encoding -> adapter -> image embedding. Stage 2: patient metadata -> tokenizer -> embedding model -> text embedding. Stage 3: concatened image embedding + text embedding -> classification model -> fracture risk index.
